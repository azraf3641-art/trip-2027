Read this first — how to use the numbers

Costs assume 28 people, a summer week (7 nights), lodging only, per person. Food is separate unless a place is marked catered. The number you should compare is per-person lodging.

Travel isn't the deciding factor. Flights anywhere here run under ~$400 round-trip and most options are drive-able, so the things that actually separate these are lodging cost, drive time, and the rules (alcohol policy, season windows, whether the group is split across buildings).

Most of these don't publish a price — group venues quote on inquiry. So every card is labelled honestly: a real number with the math where it's published, or "by inquiry" where it isn't. I'm not inventing ranges anymore.

How this works — and how to set it up (organizer)

Votes go to a Google Form; this page is just the ballot. Every Vote for this button opens a Google Form with that venue pre-selected — tap Submit and you're counted. Everyone watches the running tally on the Form's own live results page via See live results. No accounts, no database, and nothing third-party runs in your friends' browsers beyond this page itself.

Why aren't live counts printed on each card? Google lets anyone view the aggregate chart but blocks embedding it inside another page, so the honest, always-current tally is one tap away on See live results rather than baked into the cards. For ~30 friends that's the right trade for zero backend, zero secrets, zero install.

Set it up in ~10 minutes: create the Form, turn on shareable results, grab the prefill link + entry id, paste three values into the top of this file's <script>, then deploy to GitHub Pages. Exact copy-paste steps (including how to add/edit a venue and how to reset votes) are in README.md next to this file.
